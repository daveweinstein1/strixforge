Supported Hardware & Quirks

Status: Active Validation Database (2026-01-10) | Maintainer: Dave Weinstein (@daveweinstein1)

This database tracks the specific implementation quirks of Strix Halo hardware. While the Ryzen AI Max+ chip is consistent, manufacturers vary in cooling, BIOS options, and peripheral controllers (especially 10GbE and USB4). We have validated the following devices in detail.

Validated Hardware

Framework Desktop (Mainboard) Status: Validated

Config: Ryzen AI Max+ 395 | 128GB LPDDR5x

The reference standard for StrixForge. We use the desktop-class mainboard for all internal testing.

Note: Thermal Throttling
If used inside the Laptop 16 chassis, the "Performance" power profile generates significant fan noise.
StrixForge Action: We recommend using the Mainboard in a desktop/SFF enclosure for 24/7 AI workloads to maintain boost clocks without thermal constraints.
Beelink GTR9 Pro Status: Awaiting Next Hardware Revision

Config: Ryzen AI Max+ 395 | 32/64/128GB

CRITICAL: Intel E610 10GbE Crash
On Linux Kernel 6.12+, the Intel I226-V / E610 NICs cause kernel panics or "soft lockups" under load.
BIOS Fix: Navigate to Advanced → Demo Board → PCI-E Port → Device 3 Fun 2 and set to Disabled.
Kernel Fix: StrixForge automatically applies `module_blacklist=ice` to prevents the driver from loading power management features.
Minisforum MS-S1 Max Status: Untested, but Expected to Work

Config: Ryzen AI Max+ 395 | 32/64/96/128GB

Excellent connectivity options including OCuLink, but requires kernel parameters for stability.

Issue: Rear USB4 Instability
The rear USB4 ports can be unstable with high-bandwidth devices.
StrixForge Action: The installer applies the `pcie_port_pm=off` kernel parameter to GRUB to enforce stable power states on the PCIe bus.
